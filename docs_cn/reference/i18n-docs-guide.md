# 中英文文档维护指南

本站使用 Docusaurus 的多语言目录结构，中文是默认语言，英文放在独立的 `i18n/en` 目录中。

## 文档放置位置

| 语言 | 放置目录 | 示例 |
| --- | --- | --- |
| 中文 | `docs/` | `docs/core-board/rk3588-core-board/README.md` |
| 英文 | `i18n/en/docusaurus-plugin-content-docs/current/` | `i18n/en/docusaurus-plugin-content-docs/current/core-board/rk3588-core-board/README.md` |

英文文档应与中文文档保持相同的相对路径和文件名。这样用户切换到 English 后，系统会自动打开同一路径下的英文页面。

## 新增目录时的标题

如果新增目录时没有写 `_category_.json`，构建前会自动生成一个基础分类文件。你也可以手动创建 `_category_.json` 来指定更准确的侧边栏名称：

```json
{
  "label": "RK3588 核心板",
  "position": 1,
  "collapsed": false
}
```

英文目录中的 `_category_.json` 应写英文 label，例如：

```json
{
  "label": "RK3588 Core Board",
  "position": 1,
  "collapsed": false
}
```

## 自动更新流程

提交 Markdown / MDX 文档后，GitHub Actions 会执行 `npm run ensure-categories`：

1. 扫描中文 `docs/` 和英文 `i18n/en/docusaurus-plugin-content-docs/current/`。
2. 为缺少 `_category_.json` 的新目录自动生成分类文件。
3. 在 push 构建时把自动生成的分类文件提交回仓库。
4. Docusaurus 的 `sidebars.js` 使用 autogenerated sidebar，新增文档会自动显示在左侧栏。

顶栏产品导航的中英文显示由 `docusaurus.config.js` 和 `i18n/en/docusaurus-theme-classic/navbar.json` 管理；新增顶栏产品入口时，需要同时补充英文翻译。
