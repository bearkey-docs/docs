# Chinese and English Documentation Maintenance Guide

This site uses the standard Docusaurus i18n layout. Chinese is the default language, and English content is stored separately.

## Where to put documents

| Language | Directory | Example |
| --- | --- | --- |
| Chinese | `docs/` | `docs/core-board/rk3588-core-board/README.md` |
| English | `i18n/en/docusaurus-plugin-content-docs/current/` | `i18n/en/docusaurus-plugin-content-docs/current/core-board/rk3588-core-board/README.md` |

English documents should use the same relative path and filename as the Chinese document. When users switch to English, Docusaurus can open the matching English page automatically.

## Auto updates

`npm run ensure-categories` scans both language folders and creates missing `_category_.json` files before local start/build and in GitHub Actions. Because `sidebars.js` uses an autogenerated sidebar, new Markdown / MDX files appear in the sidebar automatically after the site rebuilds.

Top product navbar translations are stored in `i18n/en/docusaurus-theme-classic/navbar.json`. Add an English translation there whenever a new Chinese navbar label is introduced.
