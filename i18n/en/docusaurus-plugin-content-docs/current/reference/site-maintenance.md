# Site Maintenance Guide

The site is generated by Docusaurus from Markdown / MDX files and deployed through GitHub Actions.

## Important files

| File | Purpose |
| --- | --- |
| `docs/` | Chinese documentation content |
| `i18n/en/docusaurus-plugin-content-docs/current/` | English documentation content |
| `sidebars.js` | Autogenerated left sidebar configuration |
| `docusaurus.config.js` | Site title, domain, navbar, and deployment configuration |
| `src/css/custom.css` | Site styles |
| `.github/workflows/deploy.yml` | Build and deployment workflow |

## Add a page

1. Add a `.md` or `.mdx` file under the correct language folder.
2. Use the same relative path in both languages when an English version exists.
3. Add front matter such as `title`, `description`, `sidebar_position`, and `sidebar_label` when needed.
4. Push to GitHub. The workflow builds the site and generates missing `_category_.json` files.
